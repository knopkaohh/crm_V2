# 📅 Примеры использования производственного календаря

## Сценарий 1: Добавление нового заказа в производство

### Ситуация:
Заказ "Gintar" №12345 готов к производству. В нем 3 позиции:
1. Бирка Сатин классический - 1000 шт
2. Бирка Сатин премиум - 500 шт
3. Бирка Силикон - 300 шт

### Действия:

1. **Открыть страницу заказов**
   ```
   http://localhost:3000/orders
   ```

2. **Найти заказ №12345 в колонке "Ожидание материалов"**

3. **Перетащить заказ в колонку "В производстве"**
   - Автоматически откроется модальное окно

4. **Указать сроки для каждой позиции:**

   **Позиция 1: Бирка Сатин классический - 1000 шт**
   - Дата начала: 18.11.2024
   - Дата окончания: 21.11.2024

   **Позиция 2: Бирка Сатин премиум - 500 шт**
   - Дата начала: 18.11.2024
   - Дата окончания: 20.11.2024

   **Позиция 3: Бирка Силикон - 300 шт**
   - Дата начала: 19.11.2024
   - Дата окончания: 21.11.2024

5. **Нажать "Сохранить и продолжить"**

### Результат:
- Заказ переходит в статус "В производстве"
- Позиции появляются в календаре на соответствующие даты
- Календарь показывает:
  - 18.11: 2 позиции (Сатин классический + Сатин премиум)
  - 19.11: 3 позиции (все)
  - 20.11: 2 позиции (Сатин классический + Силикон)
  - 21.11: 2 позиции (Сатин классический + Силикон)

## Сценарий 2: Планирование загрузки производства

### Ситуация:
Нужно понять, какие дни наиболее загружены и можно ли принять новый заказ на 20.11.2024

### Действия:

1. **Открыть календарь**
   ```
   Заказы → Кнопка "Календарь"
   ```

2. **Перейти к ноябрю 2024**
   - Использовать стрелки ◀️ ▶️ для навигации

3. **Посмотреть на день 20.11**
   - Количество цветных полосок показывает число позиций
   - Если меньше 3 полосок видно - значит есть место для новых

4. **Кликнуть на день 20.11**
   - Откроется список всех позиций на этот день
   - Можно увидеть:
     * Номера заказов
     * Названия клиентов
     * Типы материалов
     * Количество штук
     * Комментарии по производству

5. **Оценить загрузку**
   - Если на день < 10 позиций = низкая загрузка ✅
   - Если 10-20 позиций = средняя загрузка ⚠️
   - Если > 20 позиций = высокая загрузка 🔴

### Результат:
Понимание загрузки производства и возможности принятия новых заказов

## Сценарий 3: Фокус на собственном производстве

### Ситуация:
Производственный отдел хочет видеть только те позиции, которые изготавливаются на собственном оборудовании (Сатин и Силикон), исключая заказы на стороннее производство.

### Действия:

1. **Открыть календарь**

2. **Нажать кнопку "Наше производство"**
   - Кнопка станет синей (активной)

3. **Календарь автоматически обновится**
   - Показываются только позиции из:
     * Сатин классический (синие полоски)
     * Сатин премиум (индиго полоски)
     * Силикон (фиолетовые полоски)
     * Нейлон (розовые полоски)

4. **Отключить фильтр**
   - Снова нажать "Наше производство"
   - Календарь покажет все позиции

### Результат:
Четкое понимание загрузки собственного производства без сторонних заказов

## Сценарий 4: Ежедневный контроль производства

### Ситуация:
Начало рабочего дня. Нужно узнать, что именно должно быть изготовлено сегодня.

### Действия:

1. **Открыть календарь**

2. **Нажать кнопку "Сегодня"**
   - Календарь автоматически откроет текущий месяц
   - Текущий день будет выделен синей рамкой

3. **Кликнуть на сегодняшний день**

4. **Просмотреть список позиций:**
   ```
   Заказ №12345 - Gintar
   └─ Бирка Сатин классический - 1000 шт
      Цвет основы: Белый
      Цвет печати: Черный
      Резка: Да

   Заказ №12346 - BrandX
   └─ Бирка Сатин премиум - 500 шт
      Цвет основы: Черный
      Цвет печати: Белый
      Резка: Нет

   ... и так далее
   ```

5. **При необходимости кликнуть на заказ**
   - Откроется полная карточка заказа
   - Можно посмотреть все детали, файлы, комментарии

### Результат:
Четкий план работы на день с полной информацией по каждой позиции

## Сценарий 5: Изменение сроков производства

### Ситуация:
Клиент попросил перенести срок выполнения заказа №12345 на 2 дня позже.

### Действия:

1. **Открыть страницу заказов**

2. **Найти заказ №12345**

3. **Кликнуть на карточку заказа**
   - Откроется детальная страница заказа

4. **Если заказ уже в статусе "В производстве":**
   - Можно вручную обновить позиции через API
   - Или создать новую версию модального окна для редактирования

5. **Альтернатива:**
   - Временно вернуть заказ в "Ожидание материалов"
   - Снова перетащить в "В производстве"
   - Указать новые даты

### Результат:
Обновленные сроки в календаре

## Полезные советы

### 🎯 Цветовое кодирование
Используйте цветовую легенду для быстрой идентификации материалов:
- Синие и фиолетовые - собственное производство
- Остальные цвета - партнерское производство или специальные материалы

### 📊 Контроль загрузки
- Старайтесь не перегружать отдельные дни
- Распределяйте заказы равномерно по неделе
- Учитывайте выходные дни

### ⚡ Быстрая навигация
- Используйте кнопку "Сегодня" для мгновенного перехода
- Фильтр "Наше производство" экономит время при планировании

### 📝 Комментарии
При установке дат производства используйте поле "Комментарии" для:
- Особых требований клиента
- Технических особенностей
- Приоритетов выполнения
