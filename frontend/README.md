# Birka CRM Frontend

Frontend приложение для CRM системы Birka Market, построенное на Next.js 14 и React.

## Требования

- Node.js 18+
- npm или yarn

## Установка и запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка переменных окружения

Создайте файл `.env.local` в корне проекта `frontend`:

```env
NEXT_PUBLIC_API_URL=http://localhost:3001/api
```

Если ваш backend работает на другом порту или домене, измените URL соответственно.

### 3. Запуск проекта

**Режим разработки:**
```bash
npm run dev
```

Приложение будет доступно на `http://localhost:3000`

**Production режим:**

Сначала соберите проект:
```bash
npm run build
```

Затем запустите:
```bash
npm start
```

## Доступные скрипты

- `npm run dev` - Запуск в режиме разработки на `http://localhost:3000`
- `npm run build` - Сборка production версии приложения
- `npm start` - Запуск собранного приложения (требует предварительной сборки)
- `npm run lint` - Проверка кода линтером

## Структура проекта

```
frontend/
├── app/              # Next.js App Router страницы
│   ├── dashboard/    # Дашборд
│   ├── clients/      # Клиенты
│   ├── leads/        # Лиды
│   ├── orders/       # Заказы
│   ├── tasks/        # Задачи
│   ├── calls/        # Звонки
│   └── login/        # Страница входа
├── components/       # React компоненты
├── lib/              # Утилиты и API клиент
└── public/           # Статические файлы
```

## Технологии

- Next.js 14 - React фреймворк
- TypeScript - типизированный JavaScript
- Tailwind CSS - стилизация
- Axios - HTTP клиент
- React Hook Form - управление формами
- Socket.io Client - WebSocket для реального времени
- Lucide React - иконки

## Примечания

- Убедитесь, что backend сервер запущен и доступен по адресу, указанному в `NEXT_PUBLIC_API_URL`
- При первом запуске выполните `npm run build` для production или используйте `npm run dev` для разработки
- Файл `.env.local` не должен быть закоммичен в репозиторий (должен быть в `.gitignore`)
