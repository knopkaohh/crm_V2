# Производственный календарь - Руководство

## Что было реализовано

Создан полнофункциональный производственный календарь для управления производством в CRM-системе.

### Основные возможности:

1. **Визуализация производственного календаря**
   - Месячный календарь с отображением позиций в производстве
   - Цветовая индикация по типам материалов
   - Подсветка текущего дня
   - Индикаторы загруженности дней (до 30 заказов одновременно)

2. **Управление сроками производства**
   - Автоматическое открытие модального окна при переходе заказа в статус "В производстве"
   - Установка даты начала и окончания производства для каждой позиции отдельно
   - Валидация дат (дата окончания не может быть раньше даты начала)

3. **Детализация по дням**
   - Просмотр всех позиций, запланированных на конкретный день
   - Отображение всех параметров позиции (материал, цвета, комментарии)
   - Быстрый переход к заказу при клике

4. **Фильтр "Наше производство"**
   - Отображение только позиций из материалов:
     * Сатин классический
     * Сатин премиум
     * Силикон
     * Нейлон

5. **Навигация**
   - Переключение между месяцами
   - Быстрый переход к текущему дню
   - Кнопка "Календарь" на главной странице заказов

## Установка

### 1. Применить миграцию базы данных

Откройте терминал в папке проекта и выполните:

\`\`\`bash
cd backend
npx prisma db push
\`\`\`

Или если используете Windows PowerShell:

\`\`\`powershell
cd backend
npm run prisma:push
\`\`\`

Если команда npm run prisma:push не существует, добавьте её в backend/package.json:

\`\`\`json
{
  "scripts": {
    ...
    "prisma:push": "prisma db push"
  }
}
\`\`\`

### 2. Перезапустить сервер

После применения миграции перезапустите backend сервер:

\`\`\`bash
cd backend
npm run dev
\`\`\`

### 3. Перезапустить frontend (если необходимо)

\`\`\`bash
cd frontend
npm run dev
\`\`\`

## Использование

### Добавление заказа в производство:

1. На странице "Заказы" перетащите заказ в колонку "В производстве"
2. Автоматически откроется модальное окно
3. Укажите дату начала и окончания производства для каждой позиции
4. Нажмите "Сохранить и продолжить"

### Просмотр календаря:

1. На странице "Заказы" нажмите кнопку "Календарь" (фиолетовая кнопка)
2. Используйте стрелки для навигации по месяцам
3. Нажмите на день с позициями для просмотра деталей
4. Включите фильтр "Наше производство" для отображения только своих позиций

### Цветовая индикация материалов:

- 🔵 Синий - Сатин классический
- 🟣 Индиго - Сатин премиум
- 🟣 Фиолетовый - Силикон
- 🌸 Розовый - Нейлон
- 🟢 Зелёный - Хлопок
- 🟡 Жёлтый - Жаккард
- 🟠 Оранжевый - Картон
- 🔷 Бирюзовый - ZIP-Lock пакет

## Технические детали

### База данных:

Добавлены поля в таблицу `order_items`:
- `productionStartDate` (DateTime, nullable)
- `productionEndDate` (DateTime, nullable)

### API Endpoints:

1. **GET /api/production-calendar**
   - Параметры:
     * `startDate` - Начальная дата периода
     * `endDate` - Конечная дата периода
     * `ourProduction` - true/false (фильтр "Наше производство")
   - Возвращает: Массив позиций заказов в производстве

2. **GET /api/production-calendar/day/:date**
   - Параметры:
     * `date` - Дата в формате YYYY-MM-DD
     * `ourProduction` - true/false
   - Возвращает: Массив позиций на указанную дату

3. **PUT /api/production-calendar/item/:itemId**
   - Тело запроса:
     ```json
     {
       "productionStartDate": "2024-11-18",
       "productionEndDate": "2024-11-21"
     }
     ```
   - Возвращает: Обновлённую позицию
