# ⚠️ ВАЖНО: Перед использованием календаря

## 🚨 Если вы получили ошибку 500

```
Failed to set production dates: AxiosError
```

### 🎯 Быстрое решение (30 секунд)

Запустите в PowerShell из корневой папки проекта:

```powershell
.\check-and-migrate.ps1
```

Затем перезапустите backend:

```powershell
cd backend
npm run dev
```

**ВСЁ!** ✅ Теперь календарь будет работать.

---

## 📖 Подробная инструкция

Если автоматический скрипт не сработал, следуйте инструкции в файле:

**👉 [FIX_ERROR_500.md](FIX_ERROR_500.md)**

---

## 🎓 Что произошло?

При создании календаря были добавлены новые поля в базу данных для хранения дат производства. Чтобы эти поля появились, нужно применить миграцию.

---

## ✅ Как проверить, что всё готово?

Запустите в папке backend:

```powershell
cd backend
npm run check-db
```

Должно появиться:

```
✅ Поля productionStartDate и productionEndDate существуют в БД
✅ База данных готова к работе с производственным календарем
```

---
