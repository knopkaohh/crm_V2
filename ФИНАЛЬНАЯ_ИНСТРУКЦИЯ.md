## 🥇 СПОСОБ 1: Через Visual Studio Code (РЕКОМЕНДУЕТСЯ)

### Шаги:
1. Откройте VS Code
2. Откройте встроенный терминал: `Ctrl + ~`
3. Выполните:

```bash

npm run migrate:sql
```

4. Дождитесь сообщения: `✅ МИГРАЦИЯ УСПЕШНО ПРИМЕНЕНА!`
5. Перезапустите backend (Ctrl+C, затем `npm run dev`)

## 🌐 СПОСОБ 4: Через NeonDB веб-консоль (100% РАБОТАЕТ!)

Это самый надежный способ для NeonDB:

### Шаги:
1. Откройте https://console.neon.tech/
2. Войдите в свой аккаунт
3. Выберите ваш проект CRM
4. Перейдите в раздел **SQL Editor**
5. Вставьте этот SQL:

```sql
ALTER TABLE "order_items" 
ADD COLUMN IF NOT EXISTS "productionStartDate" TIMESTAMP(3),
ADD COLUMN IF NOT EXISTS "productionEndDate" TIMESTAMP(3);
```

6. Нажмите **Run** или **Execute**
7. Должно появиться: `ALTER TABLE`
8. **Перезапустите ваш backend** (Ctrl+C, затем `npm run dev`)

**ГОТОВО!** ✅

После перезапуска backend:

1. Откройте http://localhost:3000/orders
2. Найдите любой заказ в статусе "Ожидание материалов"
3. Перетащите его в колонку "В производстве"
4. Должно открыться модальное окно
5. Укажите даты начала и окончания для каждой позиции

7. Нажмите кнопку "Календарь" вверху страницы
8. Вы должны увидеть производственный календарь с вашим заказом!

## 🆘 Если ВСЁ ЕЩЁ не работает

1. **Проверьте, что backend перезапущен** после миграции
2. **Очистите кэш Prisma:**

```bash
cd backend
rm -rf node_modules/.prisma
npx prisma generate
npm run dev
```

3. **Посмотрите логи backend** при попытке сохранить даты

4. **Убедитесь, что DATABASE_URL правильный:**
   - Откройте `backend/.env`
   - Проверьте строку `DATABASE_URL`
   - Должна содержать `neon.tech` в URL

5. **Проверьте NeonDB проект:**
   - Зайдите на https://console.neon.tech/
   - Убедитесь, что проект активен (не suspended)
