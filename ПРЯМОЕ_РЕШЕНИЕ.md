# 🚨 ПРЯМОЕ РЕШЕНИЕ ОШИБКИ 500

## Проблема
Миграция не применена → поля не существуют в БД → ошибка 500

## ✅ Решение за 2 минуты

### Способ 1: Через DBeaver / pgAdmin / любой SQL клиент

1. **Откройте вашу PostgreSQL базу данных** (используйте DBeaver, pgAdmin или другой SQL клиент)

2. **Выполните этот SQL запрос:**

```sql
-- Добавление полей для производственного календаря
ALTER TABLE "order_items" 
ADD COLUMN IF NOT EXISTS "productionStartDate" TIMESTAMP(3),
ADD COLUMN IF NOT EXISTS "productionEndDate" TIMESTAMP(3);
```

3. **Готово!** Перезапустите backend и ошибка исчезнет.

---

### Способ 2: Через psql (командная строка PostgreSQL)

1. Откройте CMD (не PowerShell!)

2. Подключитесь к вашей БД:
```bash
psql -U ваш_пользователь -d название_базы
```

3. Выполните:
```sql
ALTER TABLE "order_items" 
ADD COLUMN IF NOT EXISTS "productionStartDate" TIMESTAMP(3),
ADD COLUMN IF NOT EXISTS "productionEndDate" TIMESTAMP(3);
```

4. Выйдите: `\q`

---

### Способ 3: Через файл .env

Если у вас есть DATABASE_URL в `backend/.env`, например:
```
DATABASE_URL="postgresql://user:password@localhost:5432/crm_db"
```

Откройте **CMD** (не PowerShell!) и выполните:

```bash
cd backend
npx prisma db push
```

Или:

```bash
cd backend
npm run migrate
```

---

## 🔍 Проверка

После выполнения SQL проверьте, что поля добавлены:

```sql
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'order_items' 
AND column_name IN ('productionStartDate', 'productionEndDate');
```

Должно вернуть 2 строки:
```
productionStartDate | timestamp(3) without time zone
productionEndDate   | timestamp(3) without time zone
```

## 🆘 Если ничего не помогает

### Проверьте логи backend

В консоли, где запущен `npm run dev`, должны быть подробные логи:

```
Update production dates request: {
  itemId: '...',
  productionStartDate: '...',
  productionEndDate: '...'
}
```

Если видите ошибку типа:
- `column "productionStartDate" does not exist` → поля не добавлены
- `relation "order_items" does not exist` → неправильное имя таблицы
- `connect ECONNREFUSED` → БД не запущена

### Узнайте точную ошибку

В консоли backend (где запущен `npm run dev`) посмотрите полный текст ошибки и пришлите мне - я помогу!

## 📊 Структура таблицы после миграции

Таблица `order_items` должна иметь эти поля:

```
id                    | uuid
orderId               | uuid
name                  | text
quantity              | integer
price                 | numeric
...другие поля...
productionStartDate   | timestamp(3)  ← НОВОЕ
productionEndDate     | timestamp(3)  ← НОВОЕ
createdAt             | timestamp(3)
updatedAt             | timestamp(3)
```

## 💡 Почему возникла проблема?

PowerShell блокирует выполнение скриптов по политике безопасности Windows. 
Самый надежный способ - выполнить SQL напрямую в БД.
