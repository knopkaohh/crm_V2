# Документация: Генерация PDF счетов

## Обзор

Система автоматически генерирует PDF счет для клиента при создании заказа. PDF формируется на основе шаблона `backend/templates/Форма Заявки Шаблон.pdf` и автоматически скачивается в браузере.

## Когда загружается счет

PDF счет автоматически скачивается в браузере в следующих случаях:

### 1. При создании нового заказа из вкладки "Заказы"
- **Место**: `frontend/app/orders/new/page.tsx`
- **Действие**: Пользователь заполняет форму создания заказа и нажимает "Создать заказ"
- **Результат**: После успешного создания заказа автоматически генерируется и скачивается PDF счет

### 2. При создании заказа из лида (при переходе в статус "Заказ размещен")
- **Место**: `frontend/app/leads/page.tsx`
- **Действие**: Пользователь перетаскивает лид в колонку "Заказ размещен" и заполняет модальное окно создания заказа
- **Результат**: После успешного создания заказа автоматически генерируется и скачивается PDF счет

## Как работает функция

### Бэкенд

#### Функция генерации PDF
**Файл**: `backend/src/utils/generate-invoice.ts`

Функция `generateInvoicePDF()`:
1. Загружает шаблон PDF из `backend/templates/Форма Заявки Шаблон.pdf`
2. Заполняет поля шаблона данными из заказа:
   - `{Сегодняшняя дата}` → текущая дата в формате ДД.ММ.ГГГГ
   - `{Имя Фамилия клиента}` → полное имя клиента
   - `{Номер телефона клиента}` → телефон в формате `+7 916 354-92-87`
   - `{Номер заказа}` → номер заказа
   - `{Материал и размер}` → название позиции (для каждой позиции)
   - `{Количество}` → количество позиции
   - `{Сумма позиции}` → сумма позиции
   - `{Итоговое количество}` → сумма всех количеств
   - `{Итоговая Сумма позиции}` → итоговая сумма заказа
3. Возвращает готовый PDF в виде Buffer

#### Эндпоинт API
**Файл**: `backend/src/routes/orders.ts`

**Эндпоинт**: `GET /api/orders/:id/invoice`

**Параметры**:
- `id` - ID заказа

**Ответ**: PDF файл с заголовками:
- `Content-Type: application/pdf`
- `Content-Disposition: attachment; filename="invoice-{orderNumber}.pdf"`

**Права доступа**: Только авторизованные пользователи. Менеджеры могут скачивать только свои заказы.

### Фронтенд

#### Автоматическая загрузка PDF

После создания заказа фронтенд автоматически:
1. Получает ID созданного заказа
2. Делает запрос к `GET /api/orders/{id}/invoice` с `responseType: 'blob'`
3. Создает временную ссылку для скачивания
4. Автоматически инициирует скачивание файла
5. Очищает временные ресурсы

**Код** (пример из `frontend/app/orders/new/page.tsx`):
```typescript
const res = await api.post('/orders', payload)
const createdId = res.data?.id

// Автоматическая загрузка PDF счета
if (createdId) {
  try {
    const pdfResponse = await api.get(`/orders/${createdId}/invoice`, {
      responseType: 'blob'
    })
    
    const url = window.URL.createObjectURL(new Blob([pdfResponse.data], { type: 'application/pdf' }))
    const link = document.createElement('a')
    link.href = url
    link.download = `invoice-${res.data.orderNumber || 'order'}.pdf`
    document.body.appendChild(link)
    link.click()
    document.body.removeChild(link)
    window.URL.revokeObjectURL(url)
  } catch (pdfError) {
    console.error('Failed to download invoice:', pdfError)
    // Не показываем ошибку пользователю, т.к. заказ уже создан
  }
}
```

## Форматирование данных

### Формат даты
Дата форматируется в формат `ДД.ММ.ГГГГ` (например, `15.01.2025`)

### Формат телефона
Телефон автоматически форматируется из любого формата в `+7 916 354-92-87`:
- `+79163549287` → `+7 916 354-92-87`
- `79163549287` → `+7 916 354-92-87`
- `9163549287` → `+7 916 354-92-87`

### Позиции заказа
- Если позиция одна: заполняется только первая строка в шаблоне
- Если позиций несколько: заполняются все строки (максимум 4)
- Итоговые значения рассчитываются автоматически

## Ручная генерация PDF

PDF счет можно также скачать вручную:

1. Перейти на страницу заказа
2. Добавить кнопку "Скачать счет" (если она есть)
3. Или использовать прямой URL: `/api/orders/{id}/invoice`

## Обработка ошибок

- Если шаблон не найден: возвращается ошибка 500 с сообщением
- Если заказ не найден: возвращается ошибка 404
- Если нет прав доступа: возвращается ошибка 403
- Если ошибка при генерации PDF: логируется в консоль сервера, возвращается ошибка 500

На фронтенде ошибки при загрузке PDF не показываются пользователю, если заказ уже создан (чтобы не прерывать процесс).

## Структура шаблона

Шаблон должен содержать следующие поля в фигурных скобках:
- `{Сегодняшняя дата}`
- `{Имя Фамилия клиента}`
- `{Номер телефона клиента}`
- `{Номер заказа}`
- `{Материал и размер}` (может быть несколько, для каждой позиции)
- `{Количество}` (может быть несколько, для каждой позиции)
- `{Сумма позиции}` (может быть несколько, для каждой позиции)
- `{Итоговое количество}`
- `{Итоговая Сумма позиции}`

## Зависимости

**Бэкенд**:
- `pdf-lib` - для работы с PDF документами
- `pdf-parse` - для чтения PDF (опционально)

**Фронтенд**:
- `axios` - для HTTP запросов (уже установлен)

## Разработка

### Добавление новых полей в шаблон

1. Добавьте поле в шаблон PDF с фигурными скобками: `{Новое поле}`
2. Добавьте замену в функцию `generateInvoicePDF()` в файле `backend/src/utils/generate-invoice.ts`:
   ```typescript
   await replaceTextInPDF(pdfDoc, '{Новое поле}', значение, firstPage, 12);
   ```

### Изменение формата данных

Форматирование выполняется в функции `generateInvoicePDF()`:
- Формат даты: измените параметры `toLocaleDateString()`
- Формат телефона: измените функцию `formatPhoneNumber()`
- Формат сумм: измените `.toFixed(2)` на нужное количество знаков

## Примеры использования

### Программная генерация PDF (бэкенд)

```typescript
import { generateInvoicePDF } from '../utils/generate-invoice';

const order = await prisma.order.findUnique({
  where: { id: orderId },
  include: {
    client: true,
    manager: true,
    items: true,
  },
});

const pdfBuffer = await generateInvoicePDF({ order });
// pdfBuffer можно отправить как файл или сохранить
```

### Запрос PDF через API (фронтенд)

```typescript
import api from '@/lib/api';

const downloadInvoice = async (orderId: string) => {

  link.href = url;
  link.download = `invoice-${orderNumber}.pdf`;

```

## Примечания

- PDF генерируется синхронно, что может занять некоторое время для больших заказов
- Шаблон PDF должен существовать по пути `backend/templates/Форма Заявки Шаблон.pdf`
- Функция поддерживает до 4 позиций в шаблоне (можно увеличить, изменив `maxPositions`)
- Формат телефона работает корректно для российских номеров (+7)
